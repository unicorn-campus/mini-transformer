어텐션은
"검색해서 골라 섞기" 입니다

복잡한 softmax(QKᵀ/√dₖ)·V 공식은 잠깐 잊으세요. 딱 하나만 기억하면 됩니다 — 내가 찾는 것(Q)과 비슷한 것(K)에 더 많이 주목해서, 그 내용(V)을 비율대로 섞는 것. 아래에서 단어를 눌러 직접 확인해 봅시다.

1. 무슨 일을 하나 2. Q·K·V 세 역할 3. 유사도 점수(내적) 4. √dₖ로 나누기 5. softmax = 비율 6. 직접 해보기 7. 코드 8. self vs cross 9. 요약

1 어텐션이 하는 일 (30초)

이 노트북이 배우는 문제는 이렇습니다. 질문 "하늘에 먹구름이 보이면 뭐가 생각나" 를 주면 답 "비가 올 것 같아" 를 만들어야 해요. 그런데 답의 "비가" 를 만들려면 질문 속 어떤 단어를 봐야 할까요? 바로 "먹구름이" 입니다. 어텐션은 이 "어떤 단어가 어떤 단어를 봐야 하는가" 를 계산하는 장치예요.

답의 한 단어를 만들 때, 질문 속 관련 깊은 단어(먹구름이)에
더 많이 주목해서 그 정보를 끌어온다
🔎도서관 비유로 시작합니다. 도서관에서 자료를 찾을 때를 떠올려 보세요. 검색어를 넣고 → 책들의 제목표(색인)와 비교해 → 관련 있는 책의 내용을 골라 읽죠. 어텐션도 똑같이 Q(검색어) · K(색인) · V(내용) 세 가지로 움직입니다.

2 주인공 셋: Query · Key · Value

어텐션에는 딱 세 개의 재료만 있습니다. 도서관에 그대로 대응돼요.

Q · Query

🙋 검색어

"나는 지금 이런 걸 찾고 있어" — 주목하는 쪽(현재 단어)이 내미는 질문.

예) 지금 답 단어 "비가"를 만드는 쪽 = Q

K · Key

🏷️ 색인(제목표)

"나는 이런 내용을 담고 있어" — 각 단어가 내거는 이름표. Q와 얼마나 맞는지 비교당함.

예) 질문 속 "먹구름이" 등 각 단어 = K

V · Value

📦 실제 내용

골라졌을 때 실제로 꺼내 오는 정보. 관련도(비율)만큼 섞여서 결과가 됨.

예) "먹구름이"가 실제로 담은 의미 = V

💡핵심: K로는 "얼마나 관련 있나"를 재고, V로는 "무엇을 가져올까"를 정합니다. 이름표(K)와 알맹이(V)를 분리해 둔 것이 어텐션의 영리한 점이에요.
⚠️헷갈리기 쉬운 점: "질문 문장"이라고 해서 Q가 되는 게 아닙니다! Q는 "지금 정보를 찾으러 나선 쪽"이라서, 오히려 답을 만드는 단어("비가")가 Q이고 질문 문장("하늘에·먹구름이·...")은 검색당하는 K·V가 됩니다. (6번에서 직접 확인해요)

3 1단계: 얼마나 비슷한가? (내적 = QKᵀ)

Q와 K가 얼마나 맞는지를 재는 도구가 내적(dot product) 입니다. 어렵지 않아요. 두 화살표가 같은 방향이면 큰 값, 직각이면 0, 반대 방향이면 음수가 나옵니다.

■ Q (검색어) ■ K (색인) 내적 Q·K = —
🧭 방향이 거의 같을 때(0°) 점수 최대, 직각(90°)에서 0, 반대(180°) 에서 가장 낮은 음수. 즉 "의미가 통하는 단어일수록 높은 점수" — 이게 관련도를 숫자로 바꾼 것입니다.

4 2단계: 왜 √dₖ로 나눌까? (Scaled)

함수 이름의 "Scaled" 가 바로 이 단계입니다. 점수를 √dₖ(벡터 차원의 제곱근)로 나눠 줍니다. 이유가 있어요.

💥 벡터 차원(dₖ)이 커지면 내적은 여러 항을 더하므로 값이 자연스럽게 커집니다. 점수가 너무 크면 다음 단계 softmax가 한 단어에만 100%로 쏠려(포화) 나머지를 아예 안 보게 되고, 학습 기울기도 거의 사라져 버립니다.

그래서 크기를 적당히 눌러 주는 게 √dₖ 나누기 입니다. 아래에서 효과를 직접 보세요.

5 3단계: 점수를 비율로 (softmax)

이제 점수들을 "합이 100%가 되는 비율" 로 바꿉니다. 그 도구가 softmax 예요.

softmax = 큰 점수는 더 크게, 작은 점수는 더 작게 만들어
전체 합이 1(=100%) 이 되도록 정리하는 함수

결과는 "어느 단어에 몇 % 주목할지" 를 나타내는 어텐션 가중치 입니다. 그리고 마지막 4단계에서, 이 비율대로 V(내용)를 섞어(가중합) 최종 결과를 만듭니다.

Q·Kᵀ 점수→ ÷ √dₖ→ softmax → 비율(%)→ × V 섞기→ 결과(context)
🧪 (참고) masked_fill(mask==0, -inf) 는 "여기는 보지 마"인 자리를 softmax 전에 -무한대로 만들어 그 비율을 0%로 눌러 버리는 장치입니다. 패딩·미래 단어를 가릴 때 씁니다.

6 직접 해보기 — 답 단어를 눌러보세요

7 코드는 이 다섯 줄이 전부

노트북 example-detailed.ipynb 의 scaled_dot_product_attention 함수입니다. 지금까지 본 4단계가 그대로 한 줄씩 들어 있어요.

def scaled_dot_product_attention(q, k, v, mask=None, dropout=None):
    d_k = q.size(-1)                                        # 벡터 차원 dₖ
    scores = torch.matmul(q, k.transpose(-2,-1)) / math.sqrt(d_k)  # ①②: QKᵀ ÷ √dₖ
    if mask is not None:
        scores = scores.masked_fill(mask==0, float("-inf"))     # 가릴 곳은 -무한대
    attn = torch.softmax(scores, dim=-1)                   # ③: 비율(%)로
    if dropout is not None: attn = dropout(attn)
    context = torch.matmul(attn, v)                       # ④: 비율대로 V 섞기
    return context, attn
코드단계쉬운 말
matmul(q, kᵀ)①검색어 Q와 색인 K의 유사도 점수
/ sqrt(d_k)②점수가 너무 커지지 않게 눌러줌 (Scaled)
softmax(...)③점수를 합 100%의 비율로 변환
matmul(attn, v)④비율대로 내용 V를 섞어 결과 만들기

8 헷갈리기 쉬운 점: self냐 cross냐

scaled_dot_product_attention 자체는 self-attention 전용이 아닙니다. Q·K·V 셋만 받아 계산하는 공용 수학 엔진일 뿐이에요. 그런데 아래 표에 갑자기 self_attn·cross_attn 이라는 이름이 나오면 마치 새 함수처럼 보이죠 — 사실은 같은 클래스를 두 번 만들어 이름만 다르게 붙인 것입니다. 3겹으로 쌓여 있어요.

scaled_dot_product_attention
수학 엔진 (7번의 다섯 줄)→ 감싸는 클래스 MultiHeadAttention
투영 + 헤드분할 후 엔진 호출→ 두 번 생성 self_attn · cross_attn
인코더·디코더가 붙인 인스턴스 이름

실제 코드(mini_transformer.py)로 보면 이렇습니다 — self_attn과 cross_attn은 똑같은 MultiHeadAttention 클래스를 서로 다른 속성에 담아둔 것뿐이고, 안에서는 결국 같은 엔진을 호출합니다.

class MultiHeadAttention(nn.Module):
    def forward(self, query, key, value, mask=None):
        ...
        context, attn = scaled_dot_product_attention(q, k, v, mask, self.dropout)  # 결국 여기서 만남
        return self.w_o(...)

class DecoderLayer(nn.Module):
    def __init__(self, ...):
        self.self_attn  = MultiHeadAttention(...)   # 인스턴스 ①
        self.cross_attn = MultiHeadAttention(...)   # 인스턴스 ②: 같은 클래스, 다른 이름
🧩 self.self_attn 처럼 self가 두 번 나와서 헷갈릴 수 있는데, 앞의 self는 파이썬에서 "이 객체 자신"을 가리키는 키워드이고, 뒤의 self_attn은 그냥 "self-attention용"이라는 뜻으로 코드 작성자가 붙인 변수 이름입니다. 둘은 우연히 겹친 것일 뿐 서로 관계없어요.

그래서 아래 표의 "호출 예"는 사실 self.self_attn(...) / self.cross_attn(...) 를 줄여 쓴 것이고, 실제로 다른 건 어떤 텐서를 Q·K·V로 넣느냐(그리고 마스크 유무) 뿐입니다.

어디서호출 예Q / K / V 출처종류
인코더self_attn(enc_x, enc_x, enc_x)셋 다 인코더 입력 enc_x · 마스크 없음Self-Attention
디코더①self_attn(dec_x, dec_x, dec_x)셋 다 디코더 자신 dec_x · 마스크 있음(미래 가림)Masked Self-Attn
디코더②cross_attn(dec_x, enc_out, enc_out)Q=dec_x, K·V=인코더 출력 enc_outCross-Attention
🔁 Self-Attention = 문장이 자기 자신을 둘러봄(Q·K·V가 같은 곳).
Cross-Attention = 답(디코더)이 원문(인코더)을 참고함(Q만 다른 곳).
셋 다 결국 똑같은 scaled_dot_product_attention 엔진을 입력만 바꿔 재사용할 뿐입니다.
