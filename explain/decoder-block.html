Mini-Transformer · 디코더 블록 쉽게

디코더는
"답을 쓰며 곁눈질하기" 입니다

디코더 블록(DecoderLayer)은 답을 한 단어씩 써 내려가면서, ① 자기가 지금까지 쓴 답만 돌아보고 ② 문제(질문)를 곁눈질해서 다음 단어를 정합니다. 시험 볼 때 답안지 뒷장은 못 보고, 문제지는 계속 곁눈질하는 수험생과 똑같아요.

1. 무슨 일을 하나 2. 인코더와 뭐가 다를까 3. 마스크드 셀프어텐션 4. 크로스어텐션 5. 코드 6. 헷갈리기 쉬운 점 7. 요약

1 디코더 블록이 하는 일 (30초)

질문 "하늘에 먹구름이 보이면 뭐가 생각나" 를 이해한 인코더 결과가 있다고 해봅시다. 이제 답 "비가 올 것 같아" 를 한 단어씩 써야 합니다. "비가" 다음에 "올"을 쓰려면 ① 지금까지 쓴 답("비가") 을 보고, ② 질문("먹구름이" 등) 도 다시 살펴봐야겠죠. 이 두 가지 참고 + 정리를 한 겹으로 묶은 것이 DecoderLayer 입니다.

디코더 블록 한 겹 = 내가 쓴 답 돌아보기 → 질문 곁눈질하기 → 혼자 정리하기
✍️수험생 비유로 시작합니다. 서술형 답을 쓰는 학생을 떠올려 보세요. 한 글자 쓸 때마다 방금 내가 쓴 문장을 돌아보고, 문제지를 곁눈질해서 힌트를 다시 확인하죠. 단, 답안지 뒷장(아직 안 쓴 미래)은 절대 못 봅니다. 디코더 블록이 정확히 이 과정을 계산으로 흉내 냅니다.

2 인코더와 뭐가 다를까

인코더 블록(EncoderLayer, 부품 ⑤)은 질문을 "이해"만 하면 되니 하위층이 2개(셀프어텐션 → FFN)면 충분합니다. 디코더는 "생성"까지 해야 해서 하위층이 하나 더 필요합니다.

구분EncoderLayer (부품⑤)DecoderLayer (부품⑥, 지금 배우는 것)
하는 일질문을 이해답을 생성
하위층 수2개3개
1번째셀프어텐션 self_attn(x,x,x)마스크드 셀프어텐션 self_attn(x,x,x,tgt_mask)
2번째FFN크로스어텐션 cross_attn(x,enc,enc,src_mask)
3번째—FFN

💡 하위층이 하나 늘었을 뿐, 각 하위층의 패턴은 인코더와 똑같습니다 — x + 하위층(x) 로 잔차 연결한 뒤 LayerNorm 으로 정규화(Post-LN). 3번 반복할 뿐이에요.

3 원리① 마스크드 셀프어텐션 — 미래를 가리다

self_attn(x, x, x, tgt_mask) 는 지금까지 만든 답끼리 참조하는 셀프어텐션입니다. 그런데 학습할 땐 정답 문장 전체를 한 번에 넣기 때문에, 그냥 두면 "비가"를 만들 때 미래 단어 "올·것·같아"를 미리 훔쳐볼 수 있어 버립니다(정답 누설). 그래서 인과 마스크(causal mask) 로 미래 자리를 가립니다.

① 지금 만들고 있는 답 단어를 골라 보세요

② 비가 를 쓰는 시점에 어느 위치까지 볼 수 있는지 (✅=볼 수 있음, 🚫=미래라서 못 봄)

🚫 코드로는 torch.tril(torch.ones(L, L)) 한 줄, 대각선 포함 왼쪽 아래 삼각형만 True 로 만드는 것과 같습니다. 노트북 체크포인트(셀 23)도 3단어 예시에서 True 개수가 1+2+3=6 인지로 이 모양을 검증합니다.

4 원리② 크로스어텐션 — 질문을 곁눈질하다

cross_attn(x, enc, enc, src_mask) 는 이름처럼 서로 다른 곳을 잇습니다. Query = 지금 쓰는 답(x), Key·Value = 인코더가 이해한 질문(enc) 입니다. "지금 이 답 단어를 쓰려면, 질문의 어디를 봐야 하지?" 를 계산하는 것이 바로 크로스어텐션이에요.

Q

답 (x)

지금 쓰는 디코더 단어

K

질문 인코딩 (enc)

인코더가 이해해 둔 질문

V

질문 인코딩 (enc)

실제로 끌어올 정보

① 지금 쓰는 답 단어(Query) 고르기

② 비가 가(이) 질문의 각 단어를 곁눈질하는 비율

👀 "비가" 를 고르면 질문의 "먹구름이" 에 압도적으로 주목합니다 — attention.html· multi-head-attention.html 과 동일한 실제 예시 데이터로 계산했어요. scaled_dot_product_attention 함수를 self-attn과 똑같이 재사용하되, 입력만 (답, 질문, 질문)으로 바꾼 것이 크로스어텐션입니다.

5 코드는 이 forward 한 줄씩이 전부

노트북 example-detailed.ipynb 의 DecoderLayer 클래스입니다(셀 18).

class DecoderLayer(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout=0.1):
        super().__init__()
        self.self_attn = MultiHeadAttention(d_model, num_heads, dropout)   # 지금까지 쓴 답끼리
        self.cross_attn = MultiHeadAttention(d_model, num_heads, dropout)  # 답 → 질문 곁눈질
        self.ffn = PositionwiseFeedForward(d_model, d_ff, dropout)
        self.n1 = nn.LayerNorm(d_model); self.n2 = nn.LayerNorm(d_model); self.n3 = nn.LayerNorm(d_model)
        self.drop = nn.Dropout(dropout)
    def forward(self, x, enc, tgt_mask, src_mask):
        x = self.n1(x + self.drop(self.self_attn(x, x, x, tgt_mask)))        # ① 미래 가림
        x = self.n2(x + self.drop(self.cross_attn(x, enc, enc, src_mask)))  # ② 질문 곁눈질
        x = self.n3(x + self.drop(self.ffn(x)))                            # ③ 혼자 정리
        return x
순서코드쉬운 말
①self_attn(x,x,x,tgt_mask)미래+패딩을 가리고 지금까지 쓴 답끼리만 참고 (마스크드 셀프어텐션)
②cross_attn(x,enc,enc,src_mask)Q=답, K·V=인코더 출력 → 질문을 곁눈질 (크로스어텐션)
③ffn(x)각 위치를 혼자 한 번 더 곱씹기

6 헷갈리기 쉬운 점

🏗️ self_attn과 cross_attn은 같은 설계도(MultiHeadAttention), 다른 건물입니다. 둘 다 MultiHeadAttention 클래스지만 __init__ 에서 따로 만든 서로 다른 인스턴스라 가중치(wq/wk/wv/wo)가 완전히 독립적이에요.
🎭 tgt_mask ≠ src_mask. tgt_mask(= make_decoder_mask, 셀 22)는 패딩 + 미래 둘 다 가리고 self_attn에 쓰입니다. src_mask(= make_padding_mask)는 질문 쪽 패딩만 가리고 cross_attn에 쓰여요 — 질문엔 "미래"라는 개념이 없으니까요.
3️⃣ LayerNorm이 왜 3개? Post-LN 패턴은 하위층 하나마다 정규화 하나가 붙습니다. 하위층이 3개(마스크드 셀프어텐션·크로스어텐션·FFN)라서 자연스럽게 n1·n2·n3, 총 3개가 됩니다. 인코더는 하위층이 2개라 n1·n2 뿐이었죠.

7 30초 요약

"지금까지 쓴 답을 돌아보되 미래는 가리고(마스크드 셀프어텐션),
질문을 곁눈질해서(크로스어텐션),
혼자 한 번 더 정리(FFN)하는 것을
잔차+정규화 3번으로 묶은 것이 디코더 블록이다."
