Mini-Transformer · 인코더 다시 쉽게

인코더는
"질문을 여러 번 다시 읽으며 이해를 쌓는 것" 입니다

어텐션과 멀티헤드 어텐션은 "단어끼리 서로 참조하는 한 번의 계산"이었죠. 인코더는 그 계산을 여러 겹(layer) 쌓아서, 질문 문장을 "답을 만들 때 참고할 수 있는 문맥 벡터"로 바꿔주는 조립체입니다. 아래에서 층을 쌓을수록 이해가 어떻게 달라지는지 직접 눌러 봅시다.

1 왜 인코더가 필요할까? (30초)

질문 "하늘에 먹구름이 보이면 뭐가 생각나" 를 단어 임베딩으로만 바꾸면, 각 단어는 사전적인 뜻만 가진 채 뿔뿔이 흩어져 있습니다. "먹구름이"라는 벡터 혼자서는 "하늘에"와 붙어 있을 때의 맥락(하늘 이야기 중이라는 것)을 전혀 모릅니다.

단어 각각의 사전적 의미만으로는 부족하다
→ 문장 안에서 서로 참조하며 다듬은 "문맥 벡터"가 필요하다
📖노트북 코드 주석 그대로: "임베딩 + 위치인코딩으로 질문을 벡터열로 바꾼 뒤, 인코더 블록을 여러 겹 통과시켜 질문을 이해합니다." 그 결과물(문맥 벡터열)을 나중에 디코더가 크로스어텐션으로 참고합니다.

2 비유: 발표 준비하며 원고를 여러 번 다시 읽기

발표 준비를 할 때를 떠올려 보세요. 원고를 딱 한 번만 읽으면 단어 뜻 정도만 남습니다. 그런데 두 번, 세 번 다시 읽으면 "아, 이 문장이 앞 문장이랑 이어지는구나" 하고 문맥이 점점 또렷해지죠. 인코더가 하는 일이 정확히 이겁니다 — 같은 문장을 num_layers번(이 노트북은 2번) 다시 훑으며 단어들끼리 서로 참조(셀프어텐션)해 이해를 갱신합니다.

① 1차 정독

👀 단어 뜻 파악

임베딩 직후 상태. 각 단어가 거의 자기 자신 위주로만 의미를 갖고 있습니다.

② EncoderLayer 통과

🔗 이웃과 연결

셀프어텐션으로 주변 단어를 참조하고, FFN으로 혼자 한 번 더 곱씹습니다.

③ N겹 반복

🧠 문맥 벡터 완성

같은 과정을 층 수만큼 반복해, 디코더가 참고할 최종 이해를 만듭니다.

3 인코더가 하는 일, 딱 3단계

Encoder.forward 는 아주 단순합니다. 아래 순서를 그대로 따라갑니다.

단어번호 → 임베딩
(× √d_model)→ + 위치인코딩
(순서 정보)→ EncoderLayer × num_layers
(문맥 다듬기)
🏭실제 학습에 쓰는 값 (부품⑨ Transformer 기본 하이퍼파라미터): d_model=64, num_heads=4, num_layers=2, d_ff=256. 즉 이 노트북의 인코더는 EncoderLayer를 딱 2겹 쌓습니다.

4 코드: Encoder 클래스

노트북 example-detailed.ipynb 부품 ⑦ 의 실제 코드입니다.

class Encoder(nn.Module):
    def __init__(self, vocab, d_model, num_heads, num_layers, d_ff, dropout, max_len):
        super().__init__()
        self.d_model = d_model
        self.emb = nn.Embedding(vocab, d_model)                    # 단어번호 → 벡터
        self.pos = PositionalEncoding(d_model, max_len, dropout)   # 순서 정보
        self.layers = nn.ModuleList([EncoderLayer(d_model, num_heads, d_ff, dropout)
                                     for _ in range(num_layers)])         # 여러 겹 쌓기

    def forward(self, src, mask):
        x = self.pos(self.emb(src) * math.sqrt(self.d_model))   # 임베딩×√d_model + 위치
        for l in self.layers:
            x = l(x, mask)                                      # 층을 하나씩 통과
        return x
코드쉬운 말
self.emb(src)단어 번호를 d_model 차원 벡터로 (뜻 꺼내기)
* math.sqrt(self.d_model)임베딩 크기를 키워, 이어 더할 위치인코딩에 묻히지 않게 함
self.pos(...)위치 인코딩을 더해 순서 정보 주입
for l in self.layersEncoderLayer를 num_layers번 순서대로 통과
⚖️왜 √d_model을 곱할까? 임베딩 표는 학습 초기엔 값이 작게 초기화되는데, 위치인코딩(sin/cos)은 진폭이 항상 ~1입니다. 그냥 더하면 위치 정보가 단어 뜻을 덮어써 버릴 수 있어요. 그래서 임베딩에 √d_model(=√64=8)을 곱해 두 신호의 크기를 비슷하게 맞춰 준 뒤 더합니다.

5 안을 들여다보면: EncoderLayer 한 겹

Encoder가 반복해서 통과시키는 EncoderLayer(부품 ⑤) 하나는 이미 배운 부품 둘로만 이루어져 있습니다 — 멀티헤드 셀프어텐션과 FFN.

class EncoderLayer(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout=0.1):
        super().__init__()
        self.attn = MultiHeadAttention(d_model, num_heads, dropout)   # 셀프어텐션
        self.ffn = PositionwiseFeedForward(d_model, d_ff, dropout)
        self.n1 = nn.LayerNorm(d_model); self.n2 = nn.LayerNorm(d_model)
        self.drop = nn.Dropout(dropout)

    def forward(self, x, mask):
        x = self.n1(x + self.drop(self.attn(x, x, x, mask)))   # 잔차 + 정규화 ①
        x = self.n2(x + self.drop(self.ffn(x)))               # 잔차 + 정규화 ②
        return x
x (현재 이해)→ 셀프어텐션
attn(x,x,x)→ 잔차+정규화
n1(x+…)→ FFN
(혼자 곱씹기)→ 잔차+정규화
n2(x+…) → 다음 층으로
🔎여기서 Q=K=V=x — 셀프어텐션입니다. 어텐션 문서 8번에서 본 표 그대로, "질문 문장이 자기 자신을 둘러보는" 경우예요. 같은 scaled_dot_product_attention 공식을 그대로 재사용하되, Q·K·V 모두 지금 인코딩 중인 질문 문장 x 라는 점만 다릅니다.
➕잔차 연결(x + …)은 "원본을 그대로 더해두는" 안전장치입니다. 층이 깊어져도 원래 정보가 사라지지 않고, 학습 초반엔 "거의 아무것도 안 바꾸기"부터 시작할 수 있어 안정적입니다.

6 직접 해보기 — 층이 깊어질수록 셀프어텐션이 달라진다

질문 "하늘에·먹구름이·보이면·뭐가·생각나" 를 어텐션 문서와 같은 4개 의미 축(날씨현상·하늘배경·질문구조·서술)으로 나타낸, 아직 인코더에 들어가기 전(층 통과 전) 의 예시 벡터를 준비했습니다. 단어를 고르고, 1층 / 2층(실제 num_layers=2) 버튼을 눌러 그 단어의 셀프어텐션(자기 자신 포함, 질문 속 다섯 단어를 보는 비율)이 층을 통과할수록 어떻게 바뀌는지 실제 scaled_dot_product_attention 공식으로 계산해 봅니다.

① 어느 층의 셀프어텐션을 볼까요?

② 어느 단어의 셀프어텐션을 볼까요?

먹구름이 가(이) 1층 에서 질문의 각 단어(자기 자신 포함)에 주목하는 비율

👀 "하늘에"·"먹구름이"·"생각나" 처럼 이미 뜻이 뚜렷한 단어는 1층에서부터 자기 자신에 90% 이상 주목하고, 2층에서는 거의 100% 로 더 굳어집니다 — 잔차 연결 덕분에 원래 정체성이 층을 거칠수록 더 단단해지는 거예요. 반면 "보이면"·"뭐가" 처럼 같은 "질문 구조" 역할을 공유하는 단어끼리는 서로 30~50%씩 주목을 나눠 갖습니다 — 비슷한 역할의 이웃을 셀프어텐션이 실제로 참조한다는 뜻이죠.

* 실제 코드는 잔차 다음에 LayerNorm으로 정규화하지만, 이 데모는 "잔차로 이전 정보를 유지한 채 셀프어텐션 결과를 더한다"는 핵심 원리만 보여주기 위해 정규화 단계는 생략했습니다.

7 헷갈리기 쉬운 점

🎭인코더는 왜 마스크를 하나만 받을까? forward(self, src, mask) 를 보면 디코더와 달리 마스크가 하나뿐이에요. 질문 문장은 전체가 한 번에 다 보여도 되는 정보라 "미래를 가리는" 인과 마스크가 필요 없고, make_padding_mask 로 <pad> 자리만 가리면 충분합니다.
🔗이 출력, 어디에 쓰이나요? 부품⑨ Transformer.forward 를 보면 self.decoder(tgt, self.encoder(src, src_mask), tgt_mask, src_mask) 로 호출됩니다. 즉 Encoder의 출력(문맥 벡터열)이 디코더의 크로스어텐션에서 K·V로 재사용됩니다. 어텐션 문서 6번 데모에서 "먹구름이"가 답 "비가"의 주목을 받을 수 있었던 것도, 바로 이 인코더가 미리 그 벡터를 준비해 뒀기 때문이에요.
🧪이 부품만의 체크포인트 셀은 없습니다. 다른 부품과 달리 Encoder/EncoderLayer는 별도 assert 셀이 없고, 대신 부품⑨ 전체 조립과 4단계 학습 셀의 "최종 loss < 0.5" 체크포인트에서 간접적으로 정상 동작이 확인됩니다.

num_layers번(2번) 쌓아 통과시키면,
디코더가 참고할 문맥 벡터열이 완성된다."
