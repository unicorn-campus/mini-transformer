Mini-Transformer · 인코더 블록 쉽게 이해하기

인코더 블록은
"밑줄 긋고 → 혼자 곱씹고 → 원본은 안 버리는 복습 한 장" 입니다

부품③ 멀티헤드 어텐션과 부품④ FFN을 그냥 순서대로 이어붙이기만 하면, 여러 겹 쌓았을 때 원본 정보가 옅어지고 학습이 흔들립니다. EncoderLayer는 이 둘을 잔차 연결(residual) + LayerNorm이라는 안전장치로 감싸서, 안정적으로 여러 겹 쌓을 수 있는 "한 장"으로 만든 조립 단위예요.

1. 무슨 문제 2. 복습 한 장 비유 3. 코드 4. 잔차가 하는 일 5. 직접 해보기 6. 겹겹이 쌓기 7. 헷갈리기 쉬운 점 8. 요약

1 그냥 이어붙이면 안 되나요? (30초)

지금까지 만든 부품들 — 멀티헤드 어텐션(부품③), FFN(부품④) — 은 각각 "한 번의 계산"만 합니다. 그런데 실제 인코더(부품⑦)는 이 계산을 num_layers번 (기본값 2) 반복해서 겹겹이 쌓아야 문장을 깊이 이해할 수 있어요.

문제는, 어텐션 → FFN을 단순히 순서대로만 이어붙이면 층이 깊어질수록 원본 정보가 점점 옅어지고, 출력 분포가 흔들려 학습(역전파)이 잘 안 됩니다. 그래서 "한 겹"을 안전하게 설계한 게 바로 이 문서의 주인공, EncoderLayer 입니다.

📝노트북 코드 주석 그대로: "구조: 셀프어텐션 → (원본 더하기+정규화) → FFN → (더하기+정규화). 질문을 '이해'하는 한 겹."

2 비유: 원본을 지키며 복습하는 한 장

EncoderLayer 한 겹은 딱 두 가지 하위층(sublayer)으로 이루어져 있고, 각 하위층 뒤에 같은 패턴이 반복됩니다: "원본에 더하고(잔차) → 깔끔하게 정돈한다(정규화)".

① 셀프어텐션

✏️ 밑줄 긋기

문장 속 단어들이 서로 훑어보며 관련 있는 곳에 밑줄을 긋습니다(부품③ 재사용, q=k=v=x).

② 잔차+정규화

📌 원본 지키며 정리

밑줄 내용을 원래 메모(x)에 더하고, 값이 안 튀도록 다시 깔끔하게 정돈합니다(LayerNorm).

③ FFN

🤔 혼자 곱씹기

각 단어가 밑줄 내용을 바탕으로 혼자 더 깊게 생각합니다(부품④ 재사용).

④ 잔차+정규화

📌 다시 원본 지키며 정리

곱씹은 내용도 원본(①②의 결과)에 더하고 다시 정돈합니다. 이번 겹은 여기서 끝!

매 하위층마다 "원본 메모를 버리지 않고(잔차)" + "다시 깔끔하게(정규화)" 가 반복된다

3 코드 그대로 보기

실습 노트북 셀 17, 부품⑤ EncoderLayer 전체 코드입니다. 부품이 4줄(어텐션·FFN·정규화 2개·드롭아웃)뿐이라 간단해 보이지만, forward의 딱 두 줄에 안정적인 학습의 비밀이 담겨 있어요.

class EncoderLayer(nn.Module):
    def __init__(self, d_model, num_heads, d_ff, dropout=0.1):
        super().__init__()
        # 셀프어텐션: 문장 안의 단어들끼리 서로 참조합니다.
        self.attn = MultiHeadAttention(d_model, num_heads, dropout)
        self.ffn = PositionwiseFeedForward(d_model, d_ff, dropout)
        # LayerNorm 2개: 각 하위층 뒤에서 출력 분포를 정규화해 깊은 망의 학습을 안정화합니다.
        self.n1 = nn.LayerNorm(d_model); self.n2 = nn.LayerNorm(d_model)
        self.drop = nn.Dropout(dropout)

    def forward(self, x, mask):
        # 잔차 연결(x + …): 원본 입력을 그대로 더해 주어, 층이 깊어져도 기울기가 잘 흐르고
        # '아무것도 안 바꾸기'가 쉬워 학습이 안정됩니다. 그 합을 LayerNorm으로 정규화(Post-LN 방식).
        x = self.n1(x + self.drop(self.attn(x, x, x, mask)))   # 잔차 + 정규화
        # 두 번째 하위층: FFN에도 동일한 (잔차 + 정규화) 패턴을 적용.
        x = self.n2(x + self.drop(self.ffn(x)))
        return x
코드단계쉬운 말
self.attn(x, x, x, mask)①셀프어텐션 — q=k=v=x, 자기 문장끼리 밑줄 긋기
x + self.drop(...)②-잔차밑줄 결과를 원본 x에 더하기(원본 보존)
self.n1(...)②-정규화더한 값을 LayerNorm으로 다시 정돈
self.ffn(x)③각 단어가 혼자 곱씹기(position-wise)
x + self.drop(...) → self.n2(...)④①②와 동일한 (잔차+정규화) 패턴 반복

4 잔차 연결(x + …)이 하는 일

코드에서 self.attn(x,x,x,mask) 결과를 그냥 쓰지 않고, 굳이 x + self.drop(self.attn(...)) 처럼 원본 x를 다시 더합니다. 왜일까요?

🧵기울기(gradient)가 지나가는 지름길: 덧셈은 역전파에서 기울기를 그대로 통과시킵니다. 층을 아무리 깊게 쌓아도 "+x" 지름길 덕분에 맨 앞 층까지 기울기가 옅어지지 않고 잘 전달돼요.
🪞"아무것도 안 바꾸기"가 쉬워짐: 만약 셀프어텐션이 이 위치에서 딱히 배울 게 없다면, attn(x)≈0이 되도록만 학습하면 출력이 x 그대로 됩니다(항등함수). 잔차가 없으면 매번 완벽한 복제를 처음부터 다시 배워야 해서 훨씬 어려워요.

아래 5번에서는 이 "원본을 지킨다"는 게 숫자로 어떻게 드러나는지, 실제 공식으로 직접 계산해 보여드립니다.

5 직접 해보기 — 잔차를 끄면 원본이 얼마나 사라질까?

이전 문서·멀티헤드 문서에서 쓴 질문 "하늘에·먹구름이·보이면·뭐가·생각나"의 4차원 예시 벡터(K)를 그대로 인코더 입력 x로 재사용합니다. EncoderLayer의 셀프어텐션은 q=k=v=x이므로, 이 5개 벡터끼리 서로 어텐션합니다.

① 지금 살펴볼 단어(x) 고르기

② 먹구름이 의 셀프어텐션 가중치 — softmax(QKᵀ/√d_k)

③ 잔차 연결

④ 원본 x 와의 코사인 유사도 — "원본이 얼마나 남아 있나"

👀 잔차 OFF로 두면 셀프어텐션 결과(다른 단어들의 가중 평균)만 남아 원본 x와의 유사도가 뚝 떨어집니다. 잔차 ON으로 바꾸면 x를 그대로 더했으니 유사도가 다시 90%대로 뛰어오릅니다 — 이게 바로 코드 한 줄 x + self.drop(self.attn(...))이 실제로 하는 일이에요. (LayerNorm은 평균 0·표준편차 1로 크기만 정돈할 뿐, 이 방향 유사도 자체를 만드는 건 잔차입니다.)

6 이 "한 겹"을 겹겹이 쌓기 — Encoder

EncoderLayer는 딱 한 겹만 정의합니다. 여러 겹을 쌓는 건 부품⑦ Encoder가 담당해요. nn.ModuleList에 EncoderLayer를 num_layers개 담아 두고, forward에서 순서대로 통과시킵니다.

class Encoder(nn.Module):
    def __init__(self, vocab, d_model, num_heads, num_layers, d_ff, dropout, max_len):
        super().__init__()
        self.emb = nn.Embedding(vocab, d_model)
        self.pos = PositionalEncoding(d_model, max_len, dropout)
        # 같은 구조의 인코더 블록을 num_layers개 쌓습니다.
        self.layers = nn.ModuleList([EncoderLayer(d_model, num_heads, d_ff, dropout)
                                     for _ in range(num_layers)])
    def forward(self, src, mask):
        x = self.pos(self.emb(src) * math.sqrt(self.d_model))
        for l in self.layers:
            x = l(x, mask)   # EncoderLayer를 한 겹씩 통과
        return x
임베딩+위치인코딩→ EncoderLayer #1→ EncoderLayer #2→ 인코더 출력 (enc)
🏭실제 학습에 쓰는 값(부품⑨ Transformer 기본 하이퍼파라미터): d_model=64, num_heads=4, num_layers=2, d_ff=256. 즉 실전 인코더는 EncoderLayer를 딱 2겹 쌓습니다 — 이 문서의 forward 코드가 정확히 2번 실행되는 셈이에요.

7 헷갈리기 쉬운 점

🆚EncoderLayer vs DecoderLayer, 뭐가 다른가요? EncoderLayer는 하위층이 2개(셀프어텐션, FFN) → LayerNorm도 2개(n1, n2). 반면 DecoderLayer는 하위층이 3개(마스크드 셀프어텐션, 크로스어텐션, FFN) → LayerNorm 3개(n1,n2,n3). 구조 패턴(잔차+정규화 반복)은 완전히 똑같고, 하위층 개수만 다릅니다.
🎭마스크 종류도 다릅니다. EncoderLayer의 mask는 패딩 마스크 하나뿐이에요(질문 문장은 이미 다 갖고 있으니 미래를 가릴 필요가 없음). DecoderLayer의 셀프어텐션은 패딩+인과(causal) 마스크를 함께 써서 미래 단어를 가립니다.
📐Post-LN 방식임에 주의: 이 코드는 LayerNorm(x + sublayer(x)) 순서(잔차를 먼저 더하고 정규화)라 Post-LN이라 부릅니다. 최신 대형 모델 다수는 x + sublayer(LayerNorm(x)) 순서인 Pre-LN을 씁니다. 둘 다 "잔차+정규화" 큰 틀은 같고, 순서만 다릅니다.

8 30초 요약

"EncoderLayer = 셀프어텐션으로 밑줄 긋고,
잔차(+x)로 원본을 지키며,
LayerNorm으로 다시 정돈하고,
FFN으로 혼자 곱씹은 뒤 한 번 더 지키고 정돈하는 — 안전한 한 겹."
