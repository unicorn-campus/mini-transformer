Mini-Transformer · 어휘 사전 다시 쉽게

어휘 사전은
"단어 ↔ 번호 전화번호부" 입니다

컴퓨터는 글자를 못 읽습니다. 그래서 모든 단어에 번호표를 붙여 줍니다 — 딱 이거예요. 사람 이름을 전화번호로 찾듯, 단어를 번호로 바꾸고(encode) 번호를 단어로 되돌리는(decode) 작은 사전. 아래에서 문장을 직접 넣어 번호로 바뀌는 걸 눈으로 봅시다.

1. 문제 (30초) 2. 토큰화 3. 사전 만들기 4. 특수 토큰 4종 5. 양방향 사전 6. encode / decode 7. 코드 8. 요약

1 문제는 딱 하나 (30초)

모델(신경망)은 숫자만 계산할 수 있습니다. "먹구름이" 같은 글자는 그대로는 곱하기·더하기를 할 수 없어요. 그래서 계산을 시작하기 전에 단어를 숫자로 바꿔 줘야 합니다.

🔢 모든 단어에 고유 번호를 붙이자
= 그게 바로 어휘 사전(Vocab)
🙅 컴퓨터가 못 하는 것

"먹구름이" × 3 = ?
글자는 계산 불가

🙆 사전을 거치면

"먹구름이" → 6
이제 숫자라 계산 가능

📖어휘 사전이 하는 일은 딱 두 가지 — ① 단어 → 번호 (모델에 넣기 전) / ② 번호 → 단어 (모델이 뱉은 답 읽기). 전화번호부처럼 양쪽으로 찾을 수 있어야 합니다.

2 첫 단계: 문장을 단어로 쪼개기 (토큰화)

번호를 붙이려면 먼저 '단어'라는 단위가 있어야겠죠. 문장을 낱개로 자르는 걸 토큰화(tokenize)라고 합니다. 이 실습에선 가장 단순하게 공백(스페이스)으로만 자릅니다.

def tokenize(s):
    return s.strip().split()    # 공백 단위로 자르기

⚠️공백으로만 자르므로 "먹구름이"는 통째로 한 단어입니다 ("먹구름"+"이"로 안 나뉨). 실제 대형 모델은 더 똑똑한 토크나이저(BPE 등)를 쓰지만, 원리는 똑같아요 — 문장을 조각으로 나눈다.

3 사전 만들기: 중복 제거 → 정렬 → 번호 붙이기

데이터에 나온 모든 단어를 모아 사전을 만듭니다. 순서는 이렇습니다 👇

단계하는 일이유
① 모으기모든 문장의 모든 단어를 한 곳에어떤 단어가 있는지 파악
② 중복 제거 (set)같은 단어는 한 번만번호는 단어당 딱 하나
③ 정렬 (sorted)가나다 순으로 줄 세우기실행할 때마다 번호가 같도록(재현성)
④ 번호 붙이기앞에서부터 0,1,2,…단어마다 고유 번호 부여

아래는 실습 데이터(질문 6개)로 실제로 사전을 만드는 과정입니다.

질문 6개 (원본 데이터)

→ 중복 제거 후 정렬된 단어들

✨ 왜 정렬(sorted)을 할까? set은 매번 순서가 뒤죽박죽일 수 있어요. 정렬하면 언제 돌려도 "먹구름이=6"으로 번호가 고정됩니다. 저장한 모델을 다시 불러올 때 번호가 어긋나지 않게 하는 안전장치예요.

4 맨 앞자리는 예약석: 특수 토큰 4종

실제 단어에 번호를 주기 전에, 0~3번은 특수 토큰이 먼저 차지합니다. 이들은 진짜 단어가 아니라 신호(기호)예요.

0 · <pad>
길이 맞춤용 빈칸. 짧은 문장을 긴 문장에 맞춰 채울 때.
1 · <sos>
문장 시작 신호(Start Of Sentence). "여기부터 답 시작".
2 · <eos>
문장 끝 신호(End Of Sentence). "답 끝. 그만 만들어".
3 · <unk>
사전에 없는 낯선 단어(Unknown)를 대신 받는 자리.
📌왜 맨 앞(0~3)에 고정할까? 이 신호들은 어떤 데이터를 쓰든 항상 같은 번호여야 코드가 편합니다 (예: "패딩 자리(=0)는 계산에서 빼라"). 그래서 리스트 맨 앞에 예약석으로 둡니다.
특수 토큰 4개 + 실제 단어들 = 완성된 어휘 사전

5 양방향 사전: itos ↔ stoi

전화번호부를 이름→번호, 번호→이름 양쪽으로 찾듯, 사전도 두 개를 함께 만듭니다.

itos = index → string
번호로 단어 찾기. 리스트(순서 = 번호).
모델의 답(번호)을 사람이 읽을 때.
stoi = string → index
단어로 번호 찾기. 딕셔너리.
사람 문장을 모델에 넣을 때.

실제로 만들어진 질문 사전 (itos)

아래가 src_vocab.itos — 왼쪽 노란 번호가 그 단어의 ID입니다. 0~3은 특수 토큰(예약석), 4번부터 실제 단어예요.

🔁stoi는 그냥 itos를 뒤집은 것입니다. stoi = {단어: 번호 for 번호, 단어 in enumerate(itos)} — 같은 정보를 반대 방향으로 빠르게 찾으려고 하나 더 만들어 둘 뿐이에요.

6 직접 해보기: encode / decode

이제 사전을 써 봅시다. 문장을 고르면 → 토큰으로 쪼개고 → 각 단어를 사전에서 찾아 번호로 바꿉니다(encode). 그리고 번호를 다시 단어로 되돌립니다(decode).

7 코드는 이 한 클래스가 전부

노트북 example-detailed.ipynb의 부품 ② 토큰화 & 단어장 부분입니다.

# 특수 토큰 4종을 항상 0~3번에 고정
PAD, SOS, EOS, UNK = "<pad>", "<sos>", "<eos>", "<unk>"
SPECIALS = [PAD, SOS, EOS, UNK]

class Vocab:
    def __init__(self, sentences):
        # ① 모든 단어 모으기 → ② set으로 중복 제거 → ③ 가나다 정렬
        toks = sorted({t for s in sentences for t in tokenize(s)})
        # itos: 번호→단어 (특수토큰 뒤에 실제 단어를 이어 붙임)
        self.itos = SPECIALS + toks
        # stoi: 단어→번호 (itos를 뒤집은 딕셔너리)
        self.stoi = {t: i for i, t in enumerate(self.itos)}

    def encode(self, s, add_special=True):
        # 단어를 번호로. 사전에 없으면 <unk> 번호로 대체
        ids = [self.stoi.get(t, self.stoi[UNK]) for t in tokenize(s)]
        # 필요하면 앞뒤에 <sos>…<eos>를 붙여 시작/끝을 알림
        return [self.sos_id] + ids + [self.eos_id] if add_special else ids

    def decode(self, ids):
        # 번호를 단어로. 특수토큰은 안 보이게 걸러서 이어 붙임
        return " ".join(self.itos[i] for i in ids if self.itos[i] not in (PAD, SOS, EOS))
💡질문과 답은 사전을 따로 만듭니다. src_vocab(질문용)과 tgt_vocab(답용) — 쓰는 단어가 다르니까요. 그래서 노트북에서 질문 단어 14개, 답 단어 18개로 크기가 다릅니다.
