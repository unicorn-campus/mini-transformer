가장 쉬운 생각: 첫째 단어엔 0, 둘째엔 1, 셋째엔 2… 그냥 더하면 되잖아?

3 핵심 비유: 이진수 계수기(주행거리계)

자동차 주행거리계나 이진수를 떠올려 보세요. 0,1,2,3… 세어 나갈 때 각 자리(비트)의 깜빡이는 속도가 다릅니다.

🚗 맨 오른쪽 자리는 매번 바뀌고, 왼쪽으로 갈수록 점점 천천히 바뀐다
5

비트0(맨 왼쪽) = 제일 빠름 · 비트3(맨 오른쪽) = 제일 느림

✨두 가지를 확인하세요.
① 위치를 1씩 올릴 때 왼쪽 비트가 제일 자주 깜빡이고, 오른쪽 비트는 가끔만 바뀝니다.
② 어떤 위치든 비트 조합(무늬)이 세상에 딱 하나뿐입니다 → 그래서 위치를 구별할 수 있어요.

바로 이겁니다. "서로 다른 속도로 깜빡이는 여러 개의 신호를 겹쳐서 위치를 표현한다." 위치 인코딩이 하는 일이 정확히 이거예요.

4 딱 한 가지만 바꾸면 = sin·cos

이진수는 좋은데 문제가 하나 있어요. 값이 0 아니면 1로 뚝뚝 끊깁니다. 신경망은 이렇게 계단처럼 끊기는 값보다 부드럽게 이어지는 값을 훨씬 좋아해요(미분·학습에 유리).

계단처럼 깜빡이는 이진수 → 부드럽게 출렁이는 파도(sin·cos)로 바꾼 것
그게 바로 위치 인코딩!

아래에서 같은 위치 슬라이더가 이번엔 이진수 대신 부드러운 파도 값(위치 인코딩 벡터)을 만듭니다. 막대가 위(양수)/아래(음수)로 부드럽게 움직이죠. 왼쪽 칸일수록 빨리, 오른쪽 칸일수록 천천히 변합니다 — 이진수와 똑같은 원리!

position = 5

각 칸 = 벡터의 한 차원(dim0…dim7) · 위=양수, 아래=음수

🔍 이진수의 "비트"가 위치 인코딩에선 "차원(dim)"이 됩니다. 0/1 대신 -1~1 사이의 부드러운 값일 뿐, "자리마다 속도가 다르다 + 조합이 유일하다"는 성질은 그대로예요.

그럼 sin·cos 공식은?

겁먹지 마세요. 방금 본 걸 수식으로 적은 것뿐입니다.

공식 조각쉬운 말
sin / cos부드럽게 출렁이는 파도 만들기
pos지금 몇 번째 단어인지 (슬라이더 값)
10000^(2i/d)차원마다 파도 속도를 다르게 (= 비트 자리별 속도)

5 전체 그림 — 이제 히트맵이 읽힌다

먼저: 차원마다 다른 sin·cos 파도

히트맵을 보기 전에, 차원(dim)마다 파도가 얼마나 빠른지를 곡선으로 봅시다. 각 색이 하나의 차원입니다. 왼쪽 차원(dim0)일수록 빠르게, 오른쪽으로 갈수록 느리게 출렁이죠 — 이진수 비트의 깜빡임 속도와 똑같습니다.

가로축 = 위치(pos) →·세로축 = 값(-1 ~ +1)
🌊 이 곡선들을 세로로 세워서 색으로 칠하면 → 바로 아래 히트맵이 됩니다. (한 위치에서 모든 파도 값을 뽑은 게 4번 섹션의 "막대", 그걸 위→아래로 쌓은 게 히트맵)

이제: 전체를 한 장으로 (히트맵)

모든 위치(세로)와 모든 차원(가로)의 값을 한 장에 그리면 이렇게 됩니다. "부드러운 이진 계수기를 위에서 내려다본 사진"이라고 보면 돼요.

낮음(-1)높음(+1) 가로 → 차원(dim) · 세로 ↓ 위치(pos)
👀 왼쪽 차원은 세로 줄무늬가 촘촘(빨리 깜빡 = 빠른 비트), 오른쪽 차원은 넓고 완만(천천히 = 느린 비트). 그리고 모든 가로줄(위치)의 무늬가 다 다릅니다. 딱 이진수 계수기 그대로죠!

6 코드는 이 3줄이 전부

노트북 example-detailed.ipynb에서 실제로 표를 채우는 부분입니다.

# 차원마다 다른 "깜빡임 속도" (오른쪽 dim일수록 느리게)
div = torch.exp(torch.arange(0, d_model, 2).float() * (-math.log(10000.0) / d_model))

# 짝수 칸엔 sin 파도, 홀수 칸엔 cos 파도를 채운다
pe[:, 0::2] = torch.sin(pos * div)
pe[:, 1::2] = torch.cos(pos * div)

그리고 실제로 쓸 때는 단어 벡터에 그냥 더합니다.

def forward(self, x):
    return self.dropout(x + self.pe[:, :x.size(1)])
    #              ↑ 단어 의미(x) + 위치 무늬(pe) = "세 번째 단어인 '개'"
➕왜 더하기? 이어붙이면(concat) 벡터가 길어져 계산이 늘어요. 그냥 더하면 같은 크기 안에서 "의미 + 위치"가 섞이고, 나머지는 모델이 알아서 분리해 씁니다.
