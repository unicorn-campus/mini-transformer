Mini-Transformer · 디코더 다시 쉽게

Decoder는
"곁눈질하며 쓰기를 여러 겹 반복하는 것" 입니다

디코더 블록(DecoderLayer)이 "미래는 못 보고, 질문은 곁눈질하는" 한 겹의 계산이었다면, Decoder는 그 한 겹을 num_layers번(이 노트북은 2번) 쌓아 답을 만들 최종 표현으로 다듬는 조립체입니다. 인코더와 뼈대는 같지만, 매 층마다 인코딩된 질문을 함께 참고한다는 점이 다릅니다.

1 왜 Decoder가 필요할까? (30초)

디코더 블록 하나는 "지금까지 쓴 답 돌아보기 → 질문 곁눈질하기 → 혼자 정리하기"를 딱 한 번 수행합니다. 그런데 한 번의 곁눈질만으로 "먹구름이"와 "비가" 사이의 관계를 충분히 다듬을 수 있을까요? 발표 원고를 한 번 고쳐 쓰고 끝내는 사람은 없죠. 여러 번 다시 다듬어야 표현이 정교해집니다.

디코더 블록 한 겹만으로는 부족하다
→ 같은 곁눈질 과정을 여러 겹(num_layers) 쌓아 답의 표현을 다듬어야 한다
📖노트북 코드 주석 그대로: "인코더와 뼈대는 같지만, 각 층에서 인코딩된 질문(enc)을 참고해 답을 만들 표현으로 바꿉니다." 이렇게 층을 다 통과한 최종 표현을, 부품⑨ 출력층이 다음 단어 점수(logit)로 바꿔 줍니다.

2 비유: 초안을 쓰고 또 쓰며 겹겹이 다듬기

디코더 블록 문서의 수험생 비유를 이어가 봅시다. 수험생은 답을 한 번 쓱 쓰고 끝내지 않습니다. 1차 초안을 쓴 뒤, 다시 한번 자기 답과 문제지를 번갈아 보며 표현을 다듬죠. Decoder가 하는 일이 정확히 이겁니다 — 같은 디코더 블록을 num_layers번(이 노트북은 2번) 반복 통과시키며, 매번 "지금까지 쓴 답"과 "질문"을 다시 참고합니다.

① 1차 초안

✏️ 단어 뜻 + 위치

임베딩 직후 상태. 답 단어들이 아직 서로도, 질문도 깊이 참고하지 못한 상태입니다.

② DecoderLayer 통과

👀 곁눈질하며 고쳐쓰기

마스크드 셀프어텐션으로 내가 쓴 답을, 크로스어텐션으로 질문을 다시 참고합니다.

③ N겹 반복

🎯 최종 표현 완성

같은 과정을 층 수만큼 반복해, 출력층이 다음 단어를 고를 최종 표현을 만듭니다.

3 Decoder가 하는 일, 딱 3단계

Decoder.forward 도 Encoder처럼 단순합니다. 다만 인자가 하나 더 있어요.

답 단어번호 → 임베딩
(× √d_model)→ + 위치인코딩
(순서 정보)→ DecoderLayer × num_layers
(enc·tgt_mask·src_mask 함께)
🏭실제 학습에 쓰는 값 (부품⑨ Transformer 기본 하이퍼파라미터): d_model=64, num_heads=4, num_layers=2, d_ff=256. Encoder와 똑같이 DecoderLayer를 딱 2겹 쌓습니다.

4 코드: Decoder 클래스

노트북 example-detailed.ipynb 부품 ⑧ (셀 20)의 실제 코드입니다.

class Decoder(nn.Module):
    def __init__(self, vocab, d_model, num_heads, num_layers, d_ff, dropout, max_len):
        super().__init__()
        self.d_model = d_model
        self.emb = nn.Embedding(vocab, d_model)                    # 답 단어장 기준 임베딩
        self.pos = PositionalEncoding(d_model, max_len, dropout)
        self.layers = nn.ModuleList([DecoderLayer(d_model, num_heads, d_ff, dropout)
                                     for _ in range(num_layers)])         # 디코더 블록 여러 겹

    def forward(self, tgt, enc, tgt_mask, src_mask):
        x = self.pos(self.emb(tgt) * math.sqrt(self.d_model))   # 임베딩×√d_model + 위치
        for l in self.layers:
            x = l(x, enc, tgt_mask, src_mask)                     # 매 층에 enc·마스크 둘 다 전달
        return x
코드쉬운 말
self.emb(tgt)답 단어 번호를 d_model 차원 벡터로
* math.sqrt(self.d_model)Encoder와 동일한 이유로 위치인코딩에 묻히지 않게 크기 조정
self.pos(...)위치 인코딩을 더해 순서 정보 주입
l(x, enc, tgt_mask, src_mask)DecoderLayer를 num_layers번 통과, 매번 enc(질문 인코딩)를 함께 넘김

5 Encoder와 뭐가 다를까

Encoder와 Decoder는 겉모습(임베딩+위치인코딩+층쌓기)이 거의 같습니다. 하지만 셋이 다릅니다 — 넘기는 인자, 안에 쌓는 블록, 그리고 출력층과의 관계.

구분Encoder (부품⑦, 셀19)Decoder (부품⑧, 셀20, 지금 배우는 것)
단어장src_vocab (질문)tgt_vocab (답)
쌓는 블록EncoderLayer × num_layersDecoderLayer × num_layers
forward 인자(src, mask) — 마스크 1개(tgt, enc, tgt_mask, src_mask) — enc + 마스크 2개
각 층에 넘기는 것mask만enc, tgt_mask, src_mask 모두
출력층과의 관계없음weight tying (아래에서 설명)
🔗Decoder만의 특별한 점 — weight tying
부품⑨ Transformer.__init__ 을 보면 self.out.weight = self.decoder.emb.weight 라는 줄이 있습니다. 답을 만드는 입력 임베딩(Decoder.emb) 과 출력 예측(다음 단어 점수) 이 같은 단어공간(같은 텐서) 을 쓰게 묶어, 파라미터를 줄이고 일반화를 돕습니다. Encoder의 임베딩은 출력과 무관하므로 이런 공유가 없어요.

6 직접 해보기 — 층이 깊어질수록 크로스어텐션이 달라진다

디코더 블록 문서는 한 겹의 크로스어텐션만 보여줬습니다. 하지만 실제로는 1층의 마스크드 셀프어텐션+잔차를 거치며 답 단어의 표현(x) 자체가 바뀌고, 그 바뀐 x가 2층에서 다시 질문을 곁눈질(크로스어텐션)합니다. 즉 층을 쌓을수록 "질문의 어디를 볼지"가 계속 다시 계산되는 거예요. 답 단어를 고르고 층을 바꿔가며 확인해 보세요. (질문·답 벡터는 어텐션·디코더 블록 문서와 동일한 실제 예시 데이터입니다.)

① 어느 층의 크로스어텐션을 볼까요?

② 어느 답 단어(Query)를 볼까요?

비가 가(이) 1층 에서 질문의 각 단어에 주목(곁눈질)하는 비율

👀 "비가" 는 1층에서 이미 "먹구름이" 에 강하게 주목하고, 자기 정체성이 뚜렷해 2층에서도 그 주목이 더 굳어집니다. 반면 "올"·"것"·"같아" 처럼 마스크드 셀프어텐션으로 앞 단어("비가")를 많이 참고하게 된 단어는, 2층에서 크로스어텐션 분포가 1층과 살짝 달라집니다 — 셀프어텐션이 x를 바꿔 놓았기 때문이에요.

* 실제 코드는 잔차 다음에 LayerNorm으로 정규화하고 FFN도 거치지만, 이 데모는 "셀프어텐션이 바꾼 x로 크로스어텐션을 다시 계산한다"는 핵심 원리만 보여주기 위해 LayerNorm·FFN은 생략했습니다.

7 헷갈리기 쉬운 점

🎯학습 때와 추론 때, Decoder를 부르는 방식이 다릅니다. 학습(셀 26)에서는 정답 전체 dec_in 을 한 번에 넣어(티처 포싱) Decoder를 딱 한 번 호출합니다. 반면 추론(셀 29 answer 함수)에서는 model.decode_step 을 통해 Decoder를 매 스텝마다 다시 호출하되, 인코더 결과 enc 는 한 번만 계산해 재사용합니다.
📊히트맵의 재료는 어디서 나올까? Decoder가 self.layers 라는 ModuleList 로 DecoderLayer들을 들고 있기 때문에, model.decoder.layers[-1].cross_attn.last_attn_weights 처럼 맨 마지막 층의 크로스어텐션 가중치를 직접 꺼낼 수 있습니다. 이것이 바로 어텐션 히트맵을 그리는 재료예요 — Decoder가 "층들의 리스트"이기에 가능한 접근입니다.
🧪이 부품만의 체크포인트 셀은 없습니다. Encoder와 마찬가지로 별도 assert 셀은 없고, 대신 4단계 학습 셀의 "최종 loss < 0.5" 와 5단계 답 생성 성공 에서 간접적으로 정상 동작이 확인됩니다.

8 30초 요약

"답 임베딩에 √d_model을 곱하고 위치인코딩을 더한 뒤,
마스크드 셀프어텐션 + 크로스어텐션 + FFN으로 이루어진 DecoderLayer를
num_layers번(2번) 쌓아 매번 질문(enc)을 참고하면,
출력층이 다음 단어를 고를 최종 표현이 완성된다.
(이 임베딩은 출력층과 weight tying으로 공유된다.)"
