Mini-Transformer · 부품④ FFN 쉽게 다시

FFN은
"회의 끝나고 각자 책상에서 혼자 곱씹기" 입니다

멀티헤드 어텐션이 단어들끼리 정보를 섞는 단계였다면, FFN(Position-wise Feed-Forward)은 그렇게 섞인 정보를 각 단어가 혼자 더 깊이 소화하는 단계예요. 아래에서 벡터가 커졌다 작아지는 과정을 직접 눌러 보며 확인해 봅시다.

1. 무슨 문제 2. 혼자 곱씹기 비유 3. 확대→비선형→축소 4. 코드 5. 직접 해보기 6. 헷갈리기 쉬운 점 7. 요약

1 어텐션만으론 부족한 이유 (30초)

이전 문서에서 배운 멀티헤드 어텐션은 문장 속 단어들이 서로 참고하며 정보를 섞는 일을 합니다. 그런데 섞기만 하고 끝나면 어떨까요? 각 단어는 남에게서 받은 정보를 자기 나름대로 정리하고 판단할 기회가 없습니다.

어텐션 = 회의(다른 단어 의견 듣기)
→ 회의가 끝나면 각자 자리로 돌아가 혼자 곱씹어야 진짜 내 생각이 된다
📝노트북 코드 주석 그대로: "어텐션이 '단어들끼리 정보를 섞는' 단계라면, FFN은 각 단어가 '혼자 더 깊이 곱씹는' 단계. 모든 위치(단어)에 '똑같은' 작은 2층 신경망을 독립적으로 적용합니다(그래서 position-wise)."

2 비유: 같은 업무 매뉴얼로 각자 곱씹기

FFN이 하는 일은 딱 세 단계입니다. "position-wise"라는 이름은 모든 위치(단어)가 완전히 같은 신경망을 쓰지만, 서로의 결과는 절대 들여다보지 않는다는 뜻이에요 — 같은 업무 매뉴얼을 전 직원이 각자 자리에서 혼자 따라 하는 것과 같습니다.

① 확대

📖 매뉴얼 펼치기

Linear(d_model→d_ff)로 차원을 보통 4배 늘려, 생각을 펼쳐 볼 공간을 넉넉히 마련합니다.

② 판단

✂️ 필요 없는 건 버리기

ReLU로 음수는 0으로 꺼 버리고, Dropout으로 학습 중 일부를 더 꺼서 과적합을 막습니다.

③ 압축

📎 결론만 정리

Linear(d_ff→d_model)로 다시 원래 차원으로 되돌려, 다음 층(잔차 연결)과 모양을 맞춥니다.

💡핵심 제약: 모든 위치가 동일한 W1·b1·W2·b2를 공유합니다. 그래서 문장이 길어져도(단어 수가 늘어도) FFN의 파라미터 수는 늘지 않아요 — 매뉴얼은 하나, 읽는 사람만 여럿인 셈입니다.

3 구조: 확대 → 비선형 → 축소

코드 한 줄 nn.Sequential(Linear, ReLU, Dropout, Linear) 안에 이 네 단계가 순서대로 들어 있습니다.

입력
(B,S,d_model)→ Linear1 확대
(B,S,d_ff)→ ReLU
음수 제거→ Dropout
(학습 중만)→ Linear2 축소
(B,S,d_model)
📐 왜 굳이 늘렸다 줄일까? 차원이 그대로면(d_model→d_model) 두 Linear를 이어붙여도 결국 하나의 Linear와 다를 게 없어요(선형변환의 합성은 선형변환). 일부러 넓혔다가 ReLU로 비선형성을 넣고 다시 좁혀야 "단순 곱하기"를 넘어서는 진짜 표현력이 생깁니다.
🏭실제 학습에 쓰는 진짜 값 (부품⑨ Transformer 기본 하이퍼파라미터): d_model=64, d_ff=256 → 정확히 4배 확대. 이 문서의 아래 데모는 손으로 계산을 눈으로 볼 수 있게 d_model=4, d_ff=16로 축소했지만, 4배 확대라는 비율은 실전과 동일하게 맞췄습니다.
🧯이 부품 단독 체크포인트는 없습니다. FFN은 바로 다음 부품⑤ EncoderLayer(셀 17)·부품⑥ DecoderLayer(셀 18) 안에 조립되어, 두 블록의 forward가 정상 작동하는 것으로 간접 검증됩니다.

4 코드 그대로 보기

노트북 셀 16의 PositionwiseFeedForward 클래스입니다. 딱 5줄뿐이에요.

# 직관: 어텐션이 '단어들끼리 정보를 섞는' 단계라면, FFN은 각 단어가 '혼자 더 깊이 곱씹는' 단계.
# 모든 위치(단어)에 '똑같은' 작은 2층 신경망을 독립적으로 적용합니다(그래서 position-wise).
class PositionwiseFeedForward(nn.Module):
    def __init__(self, d_model, d_ff, dropout=0.1):
        super().__init__()
        # Linear(d_model→d_ff)로 차원을 크게 늘려(보통 4배) 표현력을 키우고 → ReLU로 비선형성 부여 →
        # Dropout으로 과적합 억제 → Linear(d_ff→d_model)로 원래 차원으로 되돌립니다.
        self.net = nn.Sequential(nn.Linear(d_model, d_ff), nn.ReLU(),
                                 nn.Dropout(dropout), nn.Linear(d_ff, d_model))
    def forward(self, x):
        # x: (B, S, d_model) → (B, S, d_model). 각 위치가 서로 독립적으로 같은 net을 통과합니다.
        return self.net(x)
코드단계쉬운 말
nn.Linear(d_model,d_ff)①차원을 4배로 넓혀 표현할 공간을 늘림
nn.ReLU()②음수는 전부 0으로 — 살아남은 신호만 통과
nn.Dropout(dropout)③학습 중에만 일부를 랜덤으로 더 꺼서 과적합 방지
nn.Linear(d_ff,d_model)④다시 원래 차원으로 압축
forward(x)⑤위 net을 위치마다 독립적으로 통과 (position-wise)

5 직접 해보기 — 넓혔다 좁히는 과정, 눈으로 보기

토이 문장의 단어 3개(먹구름이·보이면·뭐가)에 각각 예시 벡터(d_model=4)를 붙였습니다. 아래 단계는 실제 Linear→ReLU→Dropout→Linear 공식 그대로 계산합니다 (가중치 W1·b1·W2·b2는 학습된 값이 아니라 구조를 보여주기 위한 고정 예시 값입니다).

① 어느 단어(위치)를 볼까요?

② 입력 벡터 먹구름이 (d_model=4) — 첫 번째 값 직접 조절 0.80
③ Linear1로 확대 (d_ff=16) — 파란/보라 = 아직 ReLU 전
④ ReLU 통과 — 회색 = 음수라서 꺼짐(0)
⑤ Dropout (학습 때만 동작, 점선 테두리 = 이번에 추가로 꺼진 자리)
⑥ Linear2로 축소 (d_model=4) — 최종 출력
🔒Position-wise 증거: 아래 표에서 먹구름이의 슬라이더를 움직여도 다른 두 단어(보이면·뭐가)의 출력은 한 자리도 바뀌지 않습니다. 완전히 같은 W1·W2를 쓰지만, 단어끼리는 계산이 철저히 독립적이에요 — 어텐션이라면 한 단어를 바꾸면 다른 단어의 결과도 흔들렸겠죠.

6 헷갈리기 쉬운 점

⚖️어텐션 vs FFN, 정보가 섞이는 방향이 다릅니다.
어텐션(부품②·③)FFN(부품④)
정보 흐름위치 사이(다른 단어 참고)위치 안(자기 자신만)
파라미터Q/K/V/O 선형변환2층 신경망(Linear-ReLU-Linear)
단어 하나 바꾸면?다른 단어 출력도 영향 받음그 단어 출력만 바뀜
🧩"같은 클래스"≠"같은 가중치". EncoderLayer(셀 17)와 DecoderLayer(셀 18)는 각각 자기만의 PositionwiseFeedForward 인스턴스를 새로 만듭니다 — 층마다 W1·W2 값이 다릅니다. 공유되는 건 오직 "한 층 안에서 위치(단어)들끼리"뿐입니다. 이 둘을 헷갈리기 쉬워요.
🚦Dropout은 학습 때만! model.eval()로 전환하면 Dropout은 자동으로 꺼지고 모든 신호를 그대로 통과시킵니다. 그래서 5번 데모의 답 생성(부품⑤ 이후, 노트북 5️⃣~6️⃣) 단계에서는 Dropout이 실제로 아무 영향도 주지 않습니다 — 학습 곡선에서만 과적합 억제 역할을 합니다.

7 30초 요약

"어텐션이 단어들끼리 정보를 섞으면,
FFN은 각 단어가 혼자 그 정보를 확대→비선형→축소로 곱씹는다.
모든 위치는 같은 신경망을 공유하지만, 계산은 서로 절대 섞이지 않는다."
